
# WordTrainerApp

Тренажёр иностранных слов для Android с интервальным повторением (SRS),
квизом, статистикой и управлением собственным словарём.

## Возможности

- **Карточки** с системой Лейтнера (SRS): слова, которые вы не знаете,
  показываются чаще. Ответ кнопками «Знаю / Не знаю» или свайпом
  (вправо — знаю, влево — не знаю). Тап по карточке открывает перевод.
- **Квиз**: выбор правильного перевода из 4 вариантов, со счётом.
- **Список слов**: поиск, фильтр «только избранное», отметка избранного,
  добавление и удаление слов, индикатор прогресса SRS.
- **Статистика**: всего слов, выучено, к повторению, серия дней (стрик),
  дневная цель и точность за сегодня.
- **Направление тренировки**: слово → перевод или наоборот.
- **Озвучка**: оффлайн (Android TTS) или онлайн-фолбэк.
- **Импорт/экспорт** колоды в JSON (`[{"word":..,"translation":..}]`).
- **Тёмная тема** (следует системной).
- Языки: EN (с встроенным словарём) и UA (пустой, наполняется импортом).
- **ИИ-коуч** (иконка чата в тулбаре): отдельный экран общения с ИИ,
  независимый от тренировки слов. Внутри — свободный чат с ИИ-учителем
  (голосовой ввод и озвучка ответов), уроки-диалоги в стиле ролевых игр
  и упражнение «собери предложение» с генерацией фраз ИИ. Два режима работы:
  **модель на устройстве** (импорт GGUF-файла, инференс через llama.cpp, офлайн)
  или **локальный сервер по сети** (OpenAI-совместимый LM Studio/Ollama на ПК).
  Режим, файл модели / IP и промпт-персона настраиваются на экране коуча.

## Архитектура

- **Kotlin**, Views + ViewBinding, минимум 4 экрана через `BottomNavigationView`.
- **Room** — единое хранилище слов и дневной статистики.
- **ViewModel + StateFlow + корутины** — вся работа с диском/БД в фоне,
  UI-поток не блокируется.
- Слои: `data` (Room, репозиторий, настройки, сидинг) · `domain`
  (SRS-логика Лейтнера, перечисления) · `tts` · `ui` (фрагменты и вьюмодели).
- **ИИ-коуч** вынесен в изолированный модуль `data/coach` (Retrofit + OkHttp +
  Gson, свои настройки и история в SharedPreferences — без Room и без связи со
  словарём) и `ui/coach` (`CoachActivity` + фрагменты чата/уроков/упражнения).
- Тесты: JUnit (`LeitnerTest`, `CsvParserTest`) и инструментальный
  `WordDaoTest` (Room in-memory).

## Сборка в Android Studio

1. Откройте папку `WordTrainerApp` в Android Studio (File → Open).
2. Убедитесь, что используется **JDK 17** (Settings → Build Tools → Gradle →
   Gradle JDK) — этого требует Android Gradle Plugin 8.4.
3. Проверьте, что `local.properties` содержит корректный `sdk.dir` для вашей
   машины.
4. Дождитесь синхронизации Gradle и запустите (Run) или соберите APK
   (Build → Build APK(s)).

## On-device движок (llama.cpp)

По умолчанию приложение собирается **без** нативной части — режим ИИ «модель на
устройстве» показывает, что движок не собран, а «локальный сервер» работает сразу.
Чтобы включить инференс GGUF прямо на телефоне:

1. Установите **NDK** и **CMake** (Android Studio → SDK Manager → SDK Tools).
2. Добавьте llama.cpp сабмодулем и зафиксируйте проверенный тег:
   ```
   git submodule add https://github.com/ggerganov/llama.cpp app/src/main/cpp/llama.cpp
   cd app/src/main/cpp/llama.cpp && git checkout <проверенный тег> && cd -
   ```
3. В `app/build.gradle.kts` раскомментируйте блоки `ndk { abiFilters ... }`,
   `ndkVersion` и `externalNativeBuild { cmake { ... } }`.
4. Пересоберите. Kotlin-обёртка — `data/coach/LlamaEngine.kt`, JNI-мост и CMake —
   `app/src/main/cpp/` (`llama_bridge.cpp`, `CMakeLists.txt`).
5. На телефоне: экран коуча → настройки → «Модель на устройстве» → «Импортировать
   GGUF». Рекомендуется instruct-модель 2–3B (~1.5–2 ГБ), телефон с 6+ ГБ ОЗУ.

Примечания: сейчас сборка ограничена ABI `arm64-v8a`; декодирование в мосте —
жадное (argmax); для GPU-ускорения соберите llama.cpp с `GGML_VULKAN`. JNI-мост
ориентирован на недавний API llama.cpp (`llama_vocab_*`) — при смене тега возможны
мелкие правки имён функций.

## Где править слова

Начальный словарь EN — `app/src/main/assets/words.json`. В приложении слова
можно добавлять и импортировать прямо на вкладке «Слова».
